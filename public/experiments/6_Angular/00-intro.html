1. AngularJS

  • AngularJS extends the HTML language by declaring several new attributes called directives.
  • It declares HTML element as a part of an Angular application.
  • The name declared by the directive must match the name of an AngularJS module implemented in a separate JavaScript file.
  • The AngularJS library declares a global object called angular allows creating applications, configurations, controllers, and other AngularJS artifacts that constitute an AngularJS application.
  • Angular is broken into various modulels the way java,c# is broken in to various classes.

2. Implement the AngularJS Application (ng-App & .module)

In HTML: <html ng-app="WhiteBoardApp">

In JS : var app= angular.module("HelloWorldApp", []);

.module in JS

  • Load AngularJS library
  • AngularJS application is implemented using the .module() method.
  • The .module() method takes two arguments.
  • The first argument is a string denoting the name of the application The second argument is an array of other modules the application depends on.
  • The array can be empty if the application does not depend on other modules.
  • There can be any number of modules depending on other modules in a dependency hierarchy.

3. Declare a Controller for an HTML Area (ng-controller & .controller)

3.1 In HTML: <div ng-controller="WhiteBoardController">

  • The HTML in an AngularJS application is considered the view of a Web application.
  • AngularJS controllers are responsible for providing data and logic that the a view renders dynamically.
  • Controllers can be responsible for an entire page or a subset of a page. They can be bound to a specific HTML element which will the scope of the controller.
  • HTML outside the element bound to a controller is outside of the scope of the controller.

Implement the Controller in JS file

3.2 In JS : app.controller("WhiteBoardController", FunctionName);

  • Use the .controller() method to name and declare a function that implements the controller.
  • The .controller() method takes two arguments. The first argument is a string used to name the controller. The second argument is the function that implements the controller.

4. Controller Sends Data to View

  • The controller is responsible for controlling the view by interacting with user events, implementing client side logic, retrieving data from the client, client side navigation, and retrieving data from the server, and providing data to the client.
  • To provide data to the client browser, the controller can request an object representation of the HTML it is responsible for, a representation of its scope of influence.
  • To do this, the function that implements the controller declares a variable specifically called $scope. The function can then refer to named placeholders and objects declared in view.
  • $scope.pvalue assigns value to the pvalue property of the scope. You can think of the scope and its properties as the data which is used to render the view. The scope is the single source-of-truth for all things view related.
  • Similarly the controller can assign behavior to scope as seen by the sayHello method
