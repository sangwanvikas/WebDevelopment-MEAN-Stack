Modules - keep namespaces separately.

jQuery's global variable can be used anywhere, Once we load jQuery lib then it can be invoked anywhere. jQuery reduces the colliding probability with another library through IIFE.

What if I write module using newer version of jQuery, but workpress module uses older version of jQuery? Since I am declaring namespace and workpress is also declaring namespaces, these two separate namespaces can also collide

Assign IIFE to a variable to export functionalities of this module outside. Since the variable is outside of the IIFE, it can be used anywhere outside.

Inside this IIFE, we will create our own API, and return it.
