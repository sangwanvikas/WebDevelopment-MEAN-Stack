    })();

It means that anything declared inside thses finctions are not usable anywhere else

Downloading scripts from various sources and from different vendors is a very powerful feature that allows developers to quickly enhance Web site behaviours. There are although potential pitfals. When a JavaScript source is downloaded it is immediately executed. If a source file declares variables and functions, these are made available in the global namespace. They can be readily invoked from anywhere in other scripts. If we download scripts from various places, it is possible that different scripts declare variables or functions with the same name creating a name space collision. Scripts loaded last can potentially re-declare names declared by earlier scripts. Namespace collition can be easily avoided by creating a namespace using a common JavaScript design pattern: Immediately-Invoked Function Expressions or IIFE.
